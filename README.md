Agentic Coding RAG - 🔀 Smart Documentation Assistant
Welcome to the Agentic Coding RAG project. This system is designed to transform fragmented technical documentation—such as architecture specs, design notes, and task lists (e.g., Markdown files generated by AI coding tools)—into an accessible, interactive, and intelligent knowledge base.

Built as part of an AI development course, this project enables developers to query their project's "brain" directly. Instead of manually searching through dozens of files to find a decision or a specific guideline, you can simply ask the system.

🎯 Project Goal
To build an intelligent RAG application that aggregates project documentation into a single knowledge layer, allowing users to extract insights, track bugs, and understand architectural decisions through natural language queries.

The system utilizes an advanced architecture to handle different types of queries:

Semantic Search (Pinecone): For specific technical questions, feature explanations, and conceptual queries.

Structured Data Extraction (JSON): For queries requiring full lists, summaries, or specific architectural data.

Intelligent Router (Event-Driven): An autonomous LLM-powered router that analyzes the user's intent to decide whether to query the vector database or the structured JSON data.

🛠️ Tech Stack
LlamaIndex: Core framework for RAG development, event-driven workflows, and intelligent routing.

Cohere: * embed-multilingual-v3.0: Embedding model for high-quality semantic vectorization.

command-r-08-2024: Large Language Model (LLM) for response generation and routing decisions using the Chat API.

Pinecone: Managed vector database for fast, scalable semantic search.

Gradio: Intuitive UI for browser-based interaction.

Pydantic: Used to define strict schemas for structured data extraction.

🏗️ Project Architecture
The project is organized into three main scripts:

prepare.py (Indexing): Reads Markdown files from the bakery-rag directory, chunks the data, converts it to embeddings via Cohere, and stores it in Pinecone under the rag-project namespace.

extract_data.py (Structured Extraction): Parses documentation to identify specific items (e.g., rules, decisions, bugs) using Pydantic schemas. It saves the resulting structured insights into extracted_data.json.

agent.py (Workflow Engine): The heart of the project. It runs an event-driven Workflow using LlamaIndex. It validates user input, uses an LLM-powered router to select the optimal search path, retrieves the relevant context, and generates a precise answer for the user.

⚙️ Setup and Installation
1. Prerequisites
Python 3.10+

uv (recommended) or pip

API Keys for Cohere and Pinecone.

2. Environment Setup
Create a .env file in the project root with the following keys:

Plaintext
COHERE_API_KEY=your_cohere_api_key_here
PINECONE_API_KEY=your_pinecone_api_key_here
3. Dependencies
Install the required packages:

Bash
uv add python-dotenv llama-index llama-index-vector-stores-pinecone llama-index-embeddings-cohere llama-index-llms-cohere pinecone-client gradio pydantic llama-index-utils-workflow
4. Data Preparation
Before querying, index your data:

Vector Indexing: uv run prepare.py

Structured Extraction: uv run extract_data.py

5. Running the Agent
Start the Gradio interface:

Bash
uv run agent.py
Open the provided local URL (typically http://127.0.0.1:7860) in your browser.

🗣️ Recommended Queries
Semantic Search Path: "What is the primary color defined in the UI guidelines?" (Expects: Point-specific answer from Pinecone).

Structured Data Path: "Can you list all the open bugs?" (Expects: Organized list extracted from the JSON).
