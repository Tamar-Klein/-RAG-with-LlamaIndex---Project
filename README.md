Agentic Coding RAG - 🔀 Smart Documentation Assistant

ברוכים הבאים לפרויקט ה-RAG (Retrieval-Augmented Generation) המבוסס על מסמכי תכנון, תיעוד וארכיטקטורה (כמו קבצי .md שנוצרים על ידי כלי Agentic Coding).

הפרויקט נבנה כחלק מקורס AI, במטרה להפוך את אוסף מסמכי התיעוד של המפתחים לנגיש, אינטראקטיבי וחכם. במקום לנבור בעשרות קבצים כדי להבין "למה התקבלה החלטה X?" או "מה ההנחיה לגבי עיצוב Y?", אפשר פשוט לשאול את המערכת.

🎯 מטרת הפרויקט

בניית מערכת המסוגלת להשיב על שאלות הקשורות לפרויקט תוכנה על בסיס קבצי התיעוד שלו (Markdown).
המערכת משתמשת בארכיטקטורה מתקדמת כדי להתמודד עם אתגרי שליפת המידע:

חיפוש וקטורי-סמנטי (Pinecone): לשאלות טכניות נקודתיות ולהסברים.

חילוץ נתונים מובנים (JSON): לשאלות הדורשות רשימות שלמות (כמו "כל ההחלטות" או "כל הבאגים הפתוחים").

ניתוב חכם (Event-Driven Router): נתב אוטומטי הבוחן את שאלת המשתמש ומחליט באופן עצמאי האם לגשת למסד הנתונים הוקטורי או לקובץ הנתונים המובנה.

🛠️ טכנולוגיות בשימוש (Tech Stack)

LlamaIndex: תשתית הליבה לפיתוח ה-RAG, ניהול אירועים (Workflows) ויצירת נתב החלטות (Selectors).

Cohere:

embed-multilingual-v3.0 - מודל ההטמעה (Embeddings) להפיכת טקסט לוקטורים.

command-r-08-2024 - מודל השפה (LLM) האחראי על ניסוח התשובות ועל החלטות הניתוב בעזרת ה-Chat API.

Pinecone: מסד נתונים וקטורי לאחסון אינדקס חיפוש מהיר של המסמכים.

Gradio: יצירת ממשק משתמש (UI) נקי ונגיש בדפדפן.

Pydantic: להגדרת סכמות (Schemas) נוקשות עבור חילוץ נתונים מובנים מהטקסט.

🏗️ מבנה הפרויקט

הפרויקט מחולק לשלושה קבצי פייתון עיקריים:

prepare.py (שלב הכנת הוקטורים):
קורא את קבצי ה-Markdown מתיקיית bakery-rag, מפרק אותם למקטעים (Chunks), ממיר אותם לוקטורים בעזרת Cohere, ושומר אותם במסד הנתונים Pinecone תחת ה-Namespace rag-project.

extract_data.py (שלב חילוץ הנתונים המובנים):
עובר על כל המסמכים ומשתמש ב-LLM כדי לחלץ מהם אובייקטים בעלי מבנה מוגדר מראש (חוקים, החלטות ארכיטקטורה, ובאגים פתוחים) באמצעות תבנית Pydantic. את התוצאות הוא שומר בקובץ המקומי extracted_data.json.

agent.py (מנוע הריצה הראשי וממשק המשתמש):
מכיל את ה-Workflow מונחה-האירועים. הוא מפעיל את ממשק ה-Gradio, מקבל שאלות, בודק את תקינותן, מנתב אותן (באמצעות LLMSingleSelector) לאפיק הנכון (Pinecone או קובץ ה-JSON), מביא את ההקשר ומנסח תשובה מדויקת למשתמש.

⚙️ הוראות התקנה והרצה

1. דרישות קדם

Python 3.10+

מנהל החבילות uv (מומלץ) או pip

מפתחות API (API Keys) לשירותים הבאים:

Cohere

Pinecone

2. הגדרת משתני סביבה

צור קובץ בשם .env בתיקיית השורש של הפרויקט, והכנס לתוכו את המפתחות שלך:

COHERE_API_KEY=your_cohere_api_key_here
PINECONE_API_KEY=your_pinecone_api_key_here


3. התקנת תלויות

התקן את החבילות הנדרשות לפרויקט:

uv add python-dotenv llama-index llama-index-vector-stores-pinecone llama-index-embeddings-cohere llama-index-llms-cohere pinecone-client gradio pydantic


(הערה: פרויקט זה משתמש ב-netfree_unstrict_ssl לטיפול בתעודות אבטחה של ספק נטפרי. אם אינך משתמש בנטפרי, ניתן להסיר ייבוא זה).

4. הכנת הנתונים (הרצה חד-פעמית או בעת עדכון קבצים)

לפני שניתן לשאול שאלות, יש לאנדקס את המידע מתוך התיקייה bakery-rag.

הכנסת נתונים למסד הנתונים הוקטורי (Pinecone):

uv run prepare.py


חילוץ הנתונים לקובץ JSON מובנה:

uv run extract_data.py


5. הפעלת ה-Agent (ממשק הצ'אט)

הרץ את הקובץ הראשי כדי לפתוח את ממשק ה-Gradio:

uv run agent.py


כנס בדפדפן לכתובת שתוצג בטרמינל (בדרך כלל http://127.0.0.1:7860).

🗣️ דוגמאות לשאלות מומלצות

נסו את השאלות הבאות בממשק כדי לראות את יכולות הנתב (Router) החכם:

שאלות שיופנו לחיפוש סמנטי (Pinecone):

"What is the primary color defined in the UI guidelines?" (צפוי: תשובה נקודתית על הצבע, למשל #F4A261).

"Why was PostgreSQL chosen for the database?"

שאלות שיופנו למאגר הנתונים המובנים (JSON):

"Can you list all the open bugs?" (צפוי: רשימה של כל הבאגים שחולצו למבנה JSON).

"What are the architecture decisions made in this project?"

הגנת קלט (Validation):

הקלדת שאלה קצרה כמו "מה" תוביל לעצירה מיידית של התהליך עם הודעת שגיאה מתאימה, מבלי לבזבז קריאות ל-API.

🎓 רפלקציה ותובנות (מסקנות מהפרויקט)

שאלות מאתגרות ל-Agent: המערכת הנוכחית מתקשה בשאלות התלויות בזמן מוחלט (לדוגמה: "אילו שדות נוספו לטבלה בשבוע האחרון?"), שכן אין לנו מעקב היסטורי מפורש (Git History) או עיבוד תאריכים מורכב בקבצים, מעבר למה שחילצנו ידנית.

החלטות ה-Router: הנתב החכם עושה עבודה טובה מאוד ברוב המקרים, אך עלול להתבלבל בניסוחים כפולים (למשל, שאלה ספציפית שכוללת את המילה "רשימה").

שדרוגים עתידיים (Future Enhancements):

סנכרון אוטומטי (Webhooks) בין קבצי ה-Markdown המקומיים לבין Pinecone וה-JSON.

הוספת סכמות נוספות לחילוץ (כגון משימות קרובות במערכת).

חיבור ישיר להיסטוריית ה-Git כדי להוסיף יכולות תשובה תלויות-זמן.
